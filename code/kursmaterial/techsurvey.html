Technology survey

Given the features and bounds defined in the project proposal, I've tried to identify a few key areas in which the use of external libraries is not only helpful but mandatory for a speedy project deliverable under the time constraints of the course.

I've identified a few areas which I think are crucial to project delivery:

  • Basic infrastructure, such as an object-relational mapping for the database back-end, sessions and authentication management, some CRUD assistance and URL rewriting.
  • Feed parsing, i.e. gathering XML feeds in a variety of formats, such as Atom and RSS, most of them probably not validated against a spec.
  • Page scraping for building a link graph to use as a basis for the page rank algorithm. HTML pages are notorious for not validating. But I'd much rather not resort to regular expressions if possible.
  • Matrix computations. When calculating things such as page rank or SVD, several pretty heavy matrix computations are involved, and implementing these myself would most probably lead to horribly ill-conditioned O(nn)-algorithms.
  • AJAX. A nice to have is to abstract away this in to basic javascript calls operating on the DOM, such that I don't have to think about it. A good to have would be framework integration, such that the same functions can be used for ordinary user GETs and AJAX GETs.
  • Rich GUI components, such as drag and drop, drop down search boxes, animations and transitions etc.

Technology selection is naturally limited by the language of choice. From the outset, I've wanted to do this project in Python. Mostly since I have no previous Python experience and thought this migth be a good time to develop some. I'm halfway proficient in Ruby, so hopefully the gap to bridge will be atleast somewhat narrow.

Basic infrastructure

Scanning the Python terrain, I've decided on Django, which seems suitably mature, has a decently sized user base and is still under active development. It includes an ORM with support for both MySQL and SQLLite. It manages nice page rewrites for clean URLs and it seems to have pretty good AJAX support. It also claims to support fully automatic generation of admin interfaces, a claim to which I will remain sceptic for a while. There is also a free book (published by APress) available, which is always nice.

Feed parsing

The open source Universal Feed Parser seems to solve feed reading in a nice way. It supports most known formats and is criminally well-tested, with over 3000 unit tests. It seems to have an understandable syntax and normalizes feed data for easier scraping. It also features a few nice-to-haves, such as automatic HTML sanitation, and automatic detection of character encoding.

Page scraping

For getting data from web pages, a combination of the built-in urllib for fetching the data of the server, and Beautiful Soup to parse it and pour it in a nice tree would seem a good fit. There are several raves about it on the web. It also has an API built for subclassing and extending, if I ever run in to some far to horribly bad mark up.

Matrix computations

SciPy seems to be well-established (it is for example used in the open source web-based Matlab clone SAGE. Its module SciPy.linalg has support for matrix inversion, SVD, eigenvalue calculations and more. It also features support for the FFT and statistical tools for calculating covariances, and doing data detrending, which might come in handy. It also supports calculating linear least squares solutions out of the box, which is a nice feature to have. A down-side is that the documentation seems lacking in places, but for the methods I'm interested in, and regarding installation, there seems to be enough.

AJAX and rich GUI

There are several good libraries out there that solve these two problems. For example, prototype and script.aculo.us (or mochikit), YUI, and jQuery with jQuery UI. For now, I've decided to settle on jQuery, since it seems to support all of what I think I need and is suitably compact. But all three alternatives seem to have some Django integration, so if jQuery turns out not to work, It will hopefully be a viable alternative to refit the application with a new library.
